# Product Adapter

Use this file when the user asks for feng shui or five-phase guidance on product strategy, onboarding, UX flow, feature prioritization, activation, retention, product-market fit, roadmap, or software/application design.

## Core Rule

Use feng shui as a metaphor for user flow, friction, support, visibility, and leakage. Do not replace user research, accessibility, security, privacy, analytics, engineering feasibility, or product strategy.

## Product Intake

Ask for:

- Product type and target user.
- Main metric or goal: activation, retention, conversion, trust, revenue, usability, learning.
- User journey or funnel.
- Current friction, drop-off, or confusion.
- Constraints: engineering, compliance, data privacy, accessibility, platform, budget.
- Screens, wireframes, analytics, or journey map if available.

## Feng Shui Translation

| Feng shui concept | Product translation |
| --- | --- |
| Entrance / qi mouth | first screen, signup, landing page, first user action |
| Ming tang | room to understand value before being pushed |
| Flow | journey continuity, navigation, handoff, task completion |
| Sha qi | abrupt prompts, dark patterns, error loops, confusing copy |
| Backing | trust signals, docs, support, undo, security, reliability |
| Leakage | churn, abandoned onboarding, unclear CTA, needless steps |
| Center | core value proposition or primary workflow |

## Five-Phase Product Lens

| Phase | Product expression | Risk | Adjustment |
| --- | --- | --- | --- |
| Wood | exploration, roadmap, extensibility | sprawling scope | metal prioritization |
| Fire | activation, delight, launch moments | distraction, hype | earth reliability |
| Earth | trust, stability, habit, support | sluggishness | wood iteration |
| Metal | precision, rules, information architecture | rigidity | water discovery |
| Water | research, personalization, networks | ambiguity | earth defaults |

## Output Pattern

1. **Product reality**: user, goal, constraints, evidence.
2. **Flow diagnosis**: entry, path, center, leakage, support.
3. **Symbolic layer**: phases, yin-yang, timing, friction.
4. **Recommendations**: UX, copy, prioritization, trust, cadence.
5. **Boundaries**: no replacement for research, testing, accessibility, privacy, or engineering review.

## Forbidden Claims

Do not claim a layout, color, phase, or launch date guarantees adoption, revenue, virality, retention, or product-market fit.
