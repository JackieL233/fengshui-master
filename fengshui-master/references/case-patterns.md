## Quick Compass Reading

Use when the user gives a bearing.

Output:

1. Convert bearing with `scripts/luopan.py`.
2. Clarify what the bearing describes.
3. Explain the sector without judging it alone.
4. Ask for plan/form context before deeper advice.

Example structure:

```text
187 degrees falls in the wu / 午 mountain, commonly grouped as S2 and associated with south/fire. By itself this only tells us the sector. For feng shui judgment I still need to know whether this is the building facing, main door, bed head, or desk facing direction, plus the surrounding form.
```

## Apartment Review Without Compass

Use form-first analysis:

- Entrance clarity and first view.
- Front-back flow.
- Bedroom command position.
- Kitchen/toilet/center concerns.
- Light, ventilation, noise, clutter.
- Missing data list: north arrow, door direction, photos, building year.

Do not assign sectors.

## Bedroom Improvement

Prioritize:

1. Sleep safety and comfort.
2. Bed support and command position.
3. Door/window/beam/mirror pressures.
4. Yin quality: light, sound, electronics, clutter.
5. Optional personal direction if birth year and method are supplied.

Keep recommendations modest and concrete.

## Office Desk Placement

Read the desk like a small site:

- Back support.
- View of entrance.
- Avoid direct corridor/door rush.
- Good light without glare.
- Cable clutter and ergonomics.
- Personal favorable direction if eight mansions data is requested.

If command position and personal direction conflict, prefer command position first.

## Retail Frontage

Treat qi as customer attention and flow:

- Visibility from approach.
- Entrance friction.
- Window display and lighting.
- Path to first decision point.
- Cashier/order visibility.
- Dead corners and congestion.
- Noise, smell, queue, and staff operations.

Translate traditional language into business-environment language.

## Flying-Star Intake

Do not produce a chart until inputs are adequate.

Ask for:

- Building completion or major renovation year.
- Facing direction in degrees.
- Floor plan with north.
- Whether the unit's door differs from building facing.
- Whether the user wants natal, annual, monthly, or renovation layer.

Then state:

- Period assumption.
- Facing/sitting convention.
- Whether chart confidence is high/medium/low.

## Date Selection Intake

Ask:

- Event type.
- Candidate date range.
- People involved and whether zodiac/bazi should be considered.
- Location/time zone.
- Hard constraints.
- Whether the user has an almanac source to use.

If no almanac data exists, provide a checklist rather than pretending to calculate.

## Comparison Matrix

For choosing between properties or layouts:

| Category | Weight | Option A | Option B | Notes |
| --- | --- | --- | --- | --- |
| External form | high |  |  | roads, support, exposure |
| Entrance | high |  |  | visibility, obstruction |
| Plan regularity | medium |  |  | center, missing corners |
| Key rooms | high |  |  | bed, kitchen, study |
| Light/air/noise | high |  |  | modern environment |
| Compass fit | optional |  |  | only if reliable |
| Remedy difficulty | medium |  |  | rental/budget constraints |

Recommend the option with fewer severe form and safety issues, not merely the most symbolic positives.
